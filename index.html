Empresa: S.I. Fiscal


Puesto: Desarrollador de Software
Periodo: 2013 → 2016

En esta empresa trabaje en el desarrollo de 3 sistemas, 1 aplicación web y 2 de escritorio, las cuales describo a continuación:

1.- El primer sistema en el que trabaje fue un sistema web para el control de soportes técnicos y contables, dirigido a los clientes de los sistemas que se ofrecen en la empresa. El cual distribuía las solicitudes registradas por los clientes entre el personal, canalizándolas a sus respectivas áreas y a los respectivos agentes. El sistema generaba resultados diversos de los soportes registrados.

El desarrollo de este sistema se llevó a cabo desde el análisis hasta su mantenimiento. Lo trabaje usando ASP Classico y MySQL (con Workbrench). En la parte de MYSQL trabaje en consultas sobre tablas, tanto selección como inserción, modificación y eliminación de registros, todo realizándolo desde scripts dentro de archivos ASP. El trabajo con ASP lo ligue a jQuery para la alteración de tablas y componentes html en base a eventos del usuario sobre la página. La parte de presentación la lleve a cabo usando HTML4+CSS2+JavaScript. Para la documentación trabajé diagramas de casos de uso, diagrama E-R para la base de datos en Visual Paradigm y los requerimientos funcionales y no funcionales.

2.- En los siguientes años trabaje en el desarrollo y mantenimiento de un sistema de escritorio, de distribución masiva y de naturaleza contable. El cual cubría manejo de pólizas, trabajo con múltiples catálogos y contribuyentes, gestión de comprobantes XML y sus validaciones consumiendo el servicio del SAT, generación de reportes diversos como estado de posición financiera, estado de resultados, balanza de comprobación, auxiliares y otros más. Además de ello realizaba los cálculos de ISR e IVA, tanto de personas físicas como de personas morales, anuales y mensuales. Finalmente cumplía con todos los últimos requisitos establecidos por el SAT sobre la presentación de contabilidad electrónica (XML), como lo fueron, balanza de comprobación, registro de pólizas, catálogo de cuentas y complementarias.
El sistema permitía trabajar con múltiples usuarios y equipos en redes LAN y WAN. Y contenía un módulo que se encargaba de realizar respaldos automáticos cada cierto tiempo de la información en la base de datos.

El desarrollo de este sistema lo llevamos a cabo principalmente sobre .NET Framework 4.5.1 con C# en WindowsForms con Visual Studio 2013 y 2015, usando SQL Server 2012 Express como almacén de datos. La aplicación estaba dividida en capas y lo que correspondía a las consultas las llevaba a cabo desde la capa de datos en la aplicación C#. Igualmente trabajé consultas con Join’s, inserciones, modificaciones y eliminaciones de registros; también trabajé recursividad usando las CTE y con manipulación de creación de columnas, tablas y llaves foráneas desde C#. La versión de SQL Server que trabajamos iba embebida en la aplicación, cada vez que se instalaba y configuraba el producto, este realizaba la instalación de una instancia y de la construcción de la base de datos (todo de manera transparente para el usuario). En la capa de negocio trabaje con Linq para realizar filtros de datos y consultas diversas.
Como reporteador usé ReportViewer 11 y 12. Trabajé consumiendo el servicio ofrecido por el SAT para validación de documentos. De instalador emplee InstalShield 2013 y 2015. Para la parte de presentación use herramientas de Syncfusion. Y para la documentación trabajé diagramas de casos de uso, diagrama E-R para la base de datos en Visual Paradigm y los requerimientos funcionales y no funcionales.

3.- En los últimos meses de relación laboral, y como último proyecto realizado como persona externa a la empresa, trabajé en un sistema de escritorio que englobaría más de 30 adendas. El cuál permitía cumplir con los requisitos para agregar información comercial de más de 30 empresas (Chedraui, Liverpool, LALA, etc.), en una misma aplicación, hacia comprobantes CFDI (XML). Este sistema se validaba por internet y se revocaba la licencia de uso también por internet. Además, este sistema se actualizaba en los clientes de forma automática, desde las instalaciones de la empresa, cada vez que así se solicitaba. Este sistema estaba dividido en 2 aplicaciones de escritorio escritas en C# usando .NET Framework 4.6.1 y WindowsForms. Y usaba como almacén de datos SQL Server 2014, tanto para trabajar las operaciones del CRUD como para trabajar Triggers, Funciones, Procedimientos almacenados, Consultas y Subconsultas, empleando recursividad con las CTE de SQL. El reporteador empleado fue ReportViewer 12. En este proyecto use capas y me apoye en su totalidad con EntityFramework 6 y Linq para sustituir la capa de datos. Además, use Installshield 2015 como generador de instaladores. Y para el manejo de archivos de gran volumen que usaba en la aplicación me apoye en la API de Dropbox. Para la parte de presentación use herramientas de Syncfusion. Y para la documentación trabajé solamente diagrama E-R para la base de datos en Visual Paradigm.
